# GridWorld Flask Application — Proposal

## 專案概述 Project Overview

本專案以 **Flask** 為後端框架，開發一個互動式網格地圖（Grid World）網頁應用程式，分為兩個作業部分：

| 作業 | 目標 |
|------|------|
| HW1-1 | 建構 n×n 可互動網格地圖，支援起點、終點與障礙物設定 |
| HW1-2 | 顯示隨機策略（上下左右箭頭）並透過策略評估計算各格狀態價值 V(s) |

---

## HW1-1：網格地圖開發

### 功能說明

1. **維度選擇**：頁面頂部提供下拉選單，讓使用者選擇 `n`（5 ≤ n ≤ 9），送出後動態渲染 n×n 網格。
2. **起點設定**：第一次點擊任意空白格 → 該格變為 **綠色**，標記為 `S`。
3. **終點設定**：第二次點擊另一空白格 → 該格變為 **紅色**，標記為 `G`。
4. **障礙物設定**：後續點擊最多 **n-2** 個空白格 → 該格變為 **灰色**，標記為 `X`。
5. **重置按鈕**：清除所有設定，回到初始空白網格。
6. **狀態提示列**：頁面下方顯示目前操作模式（「請點擊起點」、「請點擊終點」、「請設定障礙物 (剩餘 k 個)」）。

### 點擊狀態機

```
初始 → [點擊] → 設定起點(S) → [點擊] → 設定終點(G) 
     → [點擊 ×(n-2)] → 設定障礙物(X) → [完成]
```

### UI 設計重點

- 網格置中顯示，格線清晰。
- 各格大小固定（60×60 px），Hover 時有高亮效果。
- 顏色對照：起點 `#4CAF50`（綠）、終點 `#F44336`（紅）、障礙物 `#9E9E9E`（灰）。

## HW1-2：策略顯示與價值評估

### 功能說明

1. **生成策略按鈕**：點擊後，後端為每個**非終點、非障礙物**的格子隨機指定一個動作（↑ ↓ ← →），並回傳至前端顯示在格子中央。
2. **策略評估**：後端執行貝爾曼期望方程迭代，計算每格的狀態價值 **V(s)**，以小數點後兩位顯示於格子右下角。
3. **顯示切換**：可勾選顯示／隱藏 V(s) 數值，方便檢視。

### 策略評估演算法

採用 **Iterative Policy Evaluation**：

```
初始化：V(s) = 0 for all s
重複直到收斂（Δ < θ = 1e-6）：
  對每個非障礙物狀態 s：
    根據當前策略 π(s) 決定下一狀態 s'
    若 s' 越界或為障礙物，則原地不動（s' = s）
    V(s) ← R(s, a) + γ · V(s')
  （R = -1 for non-terminal, γ = 0.9）
終止條件：V(G) = 0（終點價值固定為 0）
```

### API 端點設計

| Method | Endpoint | 說明 |
|--------|----------|------|
| `POST` | `/api/generate` | 送出網格設定，回傳隨機策略 + V(s) |
| `POST` | `/api/reset` | 重置所有狀態 |

### 回傳 JSON 格式範例

```json
{
  "policy": {
    "0,0": "→", "0,1": "↓", "1,2": "←"
  },
  "values": {
    "0,0": -3.21, "0,1": -2.14, "1,2": -1.00
  }
}
```

## HW1-3：最佳策略推導（Value Iteration）

### 功能說明

1. **價值迭代演算法**：使用 Bellman Optimality Equation 迭代計算每格的最佳價值 V*(s)。
2. **最佳策略推導**：收斂後，以 argmax 反推每格的最佳動作 π*(s)。
3. **最佳策略顯示**：箭頭取代隨機策略，並以金色光暈標記最佳策略格子。
4. **更新價值函數**：V(s) 更新為最佳策略下的期望回報。

### 價值迭代演算法

採用 **Value Iteration（Bellman Optimality Equation）**：

```
初始化：V(s) = 0 for all s
重複直到收斂（Δ < θ = 1e-6）：
  對每個非障礙物、非終點狀態 s：
    V(s) ← max_a [ R(s, a) + γ · V(next_state(s, a)) ]
  （R = -1 for non-terminal, γ = 0.9）

收斂後提取最佳策略：
  π*(s) = argmax_a [ R(s, a) + γ · V(next_state(s, a)) ]
```

### API 端點

| Method | Endpoint | 說明 |
|--------|----------|------|
| `POST` | `/api/optimal` | 送出網格設定，回傳最佳策略 + V*(s) |

回傳格式與 `/api/generate` 相同。
